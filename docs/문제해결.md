# 문제 해결 가이드

## 배포 관련

### 배포가 실패했다

1. **Vercel 대시보드 확인**
   - https://vercel.com/dashboard
   - 프로젝트 → 최근 배포 선택
   - 빨간색 에러 메시지 확인

2. **일반적인 에러와 해결법**

| 에러 메시지 | 원인 | 해결 방법 |
|------------|------|---------|
| `Build failed` | 코드 문법 오류 | 로컬에서 `npm run build` 실행 후 에러 확인 |
| `Cannot find module` | 패키지 미설치 | `npm install` 실행 후 `git push` |
| `SMTP connection error` | 환경변수 오류 | Vercel 대시보드에서 환경변수 재확인 |
| `Port 3000 already in use` | 포트 충돌 | 터미널에서 `lsof -i :3000` 실행 후 프로세스 종료 |

3. **재배포**
   ```bash
   git add .
   git commit -m "fix: 배포 오류 수정"
   git push origin master
   ```

## 개발 중 문제

### 로컬 개발 서버가 시작 안 된다

```bash
# 1. npm 캐시 삭제
npm cache clean --force

# 2. node_modules 재설치
rm -r node_modules
npm install

# 3. 개발 서버 재시작
npm run dev
```

### "Cannot find module X" 에러

```bash
# 패키지 재설치
npm install

# 여전히 안 되면
npm ci  # 정확한 버전으로 설치
```

### 포트 3000이 이미 사용 중

```bash
# Windows (PowerShell)
netstat -ano | findstr :3000
taskkill /PID [PID번호] /F

# Mac/Linux
lsof -i :3000
kill -9 [PID번호]
```

## 이미지 관련

### 이미지가 안 보인다

1. **파일 경로 확인**
   ```typescript
   // ✅ 올바른 경로
   <Image src="/image/logo.png" alt="로고" width={200} height={100} />
   
   // ❌ 잘못된 경로
   <Image src="image/logo.png" alt="로고" />
   ```

2. **파일 존재 확인**
   - `public/image/` 폴더에 파일이 있는가?
   - 파일명이 정확한가? (대소문자 구분)

3. **캐시 초기화**
   - 브라우저 개발자 도구: Ctrl+Shift+Del
   - "캐시된 이미지 및 파일" 삭제

## 문의 폼 관련

### 이메일이 안 온다

1. **.env.local 확인**
   ```
   SMTP_HOST=smtp.gmail.com (정확한가?)
   SMTP_PORT=465 (포트 정확한가?)
   SMTP_USER=pinp10000@gmail.com (계정 맞는가?)
   SMTP_PASS=앱비밀번호 (올바른가?)
   ```

2. **Vercel 환경변수 확인**
   - Vercel 대시보드 → Settings → Environment Variables
   - 로컬과 동일한 값이 설정되어 있는가?

3. **Gmail 앱 비밀번호 갱신**
   - https://myaccount.google.com → Security
   - App passwords → 새로 생성

4. **로컬 테스트**
   ```bash
   npm run dev
   localhost:3000에서 문의 폼 제출
   pinp10000@gmail.com 에 메일이 오는가?
   ```

## Git 관련

### 실수로 파일을 커밋했다

```bash
# 최근 커밋 취소 (파일은 유지)
git reset --soft HEAD~1

# 파일 제외 후 다시 커밋
git reset HEAD [제외할파일]
git commit -m "수정"
git push origin master
```

### 잘못된 커밋 메시지를 수정하고 싶다

```bash
# 최근 커밋 메시지만 수정
git commit --amend -m "올바른 메시지"
git push origin master --force  # ⚠️ 주의: 팀 작업일 경우 피할 것
```

## 성능 문제

### 페이지 로드가 느리다

1. **Vercel Analytics 확인**
   - Vercel 대시보드 → Analytics
   - 느린 페이지 확인

2. **로컬 성능 측정**
   ```bash
   npm run build
   npm run start
   # 브라우저 개발자 도구 → Network 탭에서 로드 시간 확인
   ```

3. **최적화 방법**
   - 이미지 크기 줄이기
   - 불필요한 라이브러리 제거
   - Code splitting 적용
