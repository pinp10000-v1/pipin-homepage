## 로컬 개발 서버

### 서버가 시작되지 않는다 / 다운됐다

```bash
# 1. .next 캐시 삭제 후 재시작 (가장 빠른 해결)
rm -rf .next
npm run dev
```

```bash
# 2. 그래도 안 되면 node_modules 재설치
rm -rf node_modules
npm install
npm run dev
```

### 포트 충돌 (Port already in use)

Next.js가 자동으로 3001, 3002, 3003... 순서로 변경합니다.  
터미널 출력에서 실제 포트 확인 후 접속하세요.

```bash
# Windows: 특정 포트 강제 종료
taskkill /IM node.exe /F
```

## 빌드 에러

### `npm run build` 실패

| 에러 메시지 | 원인 | 해결 |
|------------|------|------|
| `Cannot find module` | import 경로 오류 | 파일 경로 및 대소문자 확인 |
| `Type error` | TypeScript 타입 불일치 | 에러 줄 확인 후 타입 수정 |
| `Image is missing required prop` | Next.js Image 필수 속성 없음 | `width`, `height` 또는 `fill` 추가 |
| `Another next build is running` | 이전 빌드 미완료 | 잠시 대기 후 재실행 |

## 이미지

### 이미지가 보이지 않는다

1. **경로 확인** — `/` 로 시작해야 함
   ```html
   ✅ <img src="/image/logo.png" />
   ❌ <img src="image/logo.png" />
   ```

2. **파일 존재 확인** — `public/image/` 폴더에 있는지 확인

3. **Next.js Image vs img 태그**  
   현재 이 프로젝트는 `<img>` 태그 직접 사용 방식 채택  
   (`next.config.js`에서 `unoptimized: true` 설정됨)

4. **.next 캐시 삭제** 후 dev 서버 재시작

## 상담 폼 (Contact)

### 이메일이 오지 않는다

1. `.env.local` 파일 확인 (보안-가이드.md 참조)
2. Vercel 환경변수 설정 확인
   - Vercel 대시보드 → Settings → Environment Variables
3. Gmail 앱 비밀번호 유효 여부 확인

## 배포

### Vercel 배포가 실패했다

1. 로컬에서 `npm run build` 먼저 실행 → 에러 확인
2. 에러 수정 후 `git push origin master`
3. Vercel 대시보드 → 해당 배포 → 로그 탭에서 상세 에러 확인

### pip-biz.com에 반영이 안 된다

- git push 후 1~2분 대기
- Vercel 대시보드에서 배포 상태(●) 확인
- 브라우저 강제 새로고침: `Ctrl + Shift + R`

## Git

### 실수로 올린 파일 취소

```bash
# 최근 커밋 취소 (파일은 유지)
git reset --soft HEAD~1

# 특정 파일만 스테이징 취소
git reset HEAD [파일명]
git commit -m "수정된 커밋"
git push origin master
```

### `.env.local`을 실수로 커밋했다

```bash
git rm --cached .env.local
git commit -m "fix: env 파일 git 추적 제거"
git push origin master
# → Gmail, Vercel API 키 즉시 재발급 필요
```

   - `npm run build` 출력 결과
