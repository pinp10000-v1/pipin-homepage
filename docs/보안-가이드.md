## 환경변수 (.env.local)

로컬 개발 시 프로젝트 루트에 `.env.local` 파일 생성:

```env
# 상담 폼 이메일 발송 (Gmail SMTP)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_USER=pinp10000@gmail.com
SMTP_PASS=Gmail_앱_비밀번호_16자리

# Google Sheets 연동 (선택)
GOOGLE_SHEETS_ID=시트ID
GOOGLE_SERVICE_ACCOUNT_EMAIL=서비스계정@project.iam.gserviceaccount.com
GOOGLE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
```

⚠️ `.env.local`은 **절대 GitHub에 올리지 말 것** (`.gitignore`에 자동 등록됨)

---

## Vercel 환경변수 설정

로컬의 `.env.local` 값을 Vercel에도 동일하게 설정해야 배포 환경에서 작동:

1. https://vercel.com/dashboard 접속
2. 프로젝트 선택 → **Settings** → **Environment Variables**
3. 위 `.env.local` 항목 동일하게 입력
4. **Save** 후 재배포 필요 (자동 적용)

---

## Gmail 앱 비밀번호 설정

일반 Gmail 비밀번호는 SMTP에서 작동하지 않음. 앱 비밀번호 생성 필요:

1. https://myaccount.google.com 접속
2. **보안** → **2단계 인증** 활성화 (필수)
3. **앱 비밀번호** → 앱: 메일 / 기기: Windows 컴퓨터
4. 생성된 **16자리 비밀번호** → `.env.local`의 `SMTP_PASS`에 입력

## 보안 규칙

### ✅ 해야 할 것
- API 키, 비밀번호는 `.env.local` 또는 Vercel 환경변수에만 저장
- 서버 코드(`app/api/`)에서만 `process.env` 접근
- 정기적으로 앱 비밀번호 교체 (3~6개월 권장)

### ❌ 하면 안 될 것
- 비밀번호를 코드에 직접 작성
- `.env.local`을 GitHub에 커밋
- 클라이언트 컴포넌트(`'use client'`)에서 서버 환경변수 접근
- 로그에 민감한 정보 출력
