# 보안 & 환경변수 관리

## 환경변수 (.env.local)

### 로컬 개발 환경 설정

`.env.local` 파일에 다음 정보 추가:

```
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_USER=pinp10000@gmail.com
SMTP_PASS=pglywjafnfgnoedi
```

**⚠️ 중요**: `.env.local`은 절대 GitHub에 올리지 말 것 (`.gitignore`에 등록됨)

## Vercel 환경변수 설정

배포 서버에 환경변수 설정:

1. https://vercel.com/dashboard 접속
2. 프로젝트 선택 → **Settings** → **Environment Variables**
3. 위의 SMTP 정보 입력
4. 배포 후 자동 반영

## 민감한 정보 보호 규칙

### ✅ 해야 할 것
- API 키, 비밀번호는 `.env.local` 또는 Vercel 환경변수에만 저장
- 환경변수 접근은 서버 사이드에서만 (예: `app/api/` 라우트)
- 클라이언트 사이드 코드에서는 `process.env.PUBLIC_` 프리픽스 변수만 사용

### ❌ 하면 안 될 것
- 비밀번호를 코드에 직접 작성
- 민감한 정보를 GitHub 커밋
- 개인키/토큰을 클라이언트 JS에 포함
- 로그 파일에 민감한 정보 기록

## 이메일 설정 (Gmail)

### Gmail 앱 비밀번호 생성

1. https://myaccount.google.com 접속
2. **Security** → **App passwords**
3. 비밀번호 입력 후 "Mail" 및 "Windows Computer" 선택
4. 생성된 16자 비밀번호를 `.env.local`의 `SMTP_PASS`에 저장

**참고**: 일반 Gmail 비밀번호는 작동하지 않음

## 문제 해결

### 이메일이 안 보낼 때
1. `.env.local` 확인: SMTP 정보가 맞는가?
2. Gmail 앱 비밀번호 갱신
3. Vercel 환경변수 재설정 후 재배포

### 보안 경고가 나올 때
- GitHub에서 "Secret scanning alerts" 확인
- 노출된 비밀번호 즉시 변경 (Gmail, Vercel)
- 새 앱 비밀번호 생성 및 업데이트
