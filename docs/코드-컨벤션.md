# 코드 작성 컨벤션

## TypeScript & React

### 컴포넌트 작성 규칙

```typescript
// ✅ Good: 함수형 컴포넌트 + Props 인터페이스
interface HeroProps {
  title: string;
  subtitle: string;
}

export default function Hero({ title, subtitle }: HeroProps) {
  return (
    <section className="bg-blue-600 text-white">
      <h1>{title}</h1>
      <p>{subtitle}</p>
    </section>
  );
}

// ❌ Bad: any 타입, props 검증 없음
export default function Hero(props: any) {
  return <h1>{props.title}</h1>;
}
```

## Tailwind CSS

### 클래스 작성 규칙

```jsx
// ✅ Good: Tailwind 유틸리티만 사용
<div className="flex items-center justify-between p-4 bg-gray-100 rounded-lg">
  <h2 className="text-xl font-bold text-gray-900">제목</h2>
</div>

// ❌ Bad: 커스텀 CSS 인라인
<div style={{ display: 'flex', backgroundColor: '#f3f4f6' }}>
  <h2 style={{ fontSize: '20px', fontWeight: 'bold' }}>제목</h2>
</div>
```

### 반응형 디자인

```jsx
// ✅ Mobile First 접근
<div className="text-sm md:text-base lg:text-lg p-4 md:p-6 lg:p-8">
  반응형 텍스트와 여백
</div>
```

## 파일 및 변수명

| 대상 | 규칙 | 예시 |
|------|------|------|
| 컴포넌트 파일 | PascalCase | `Navigation.tsx`, `ContactForm.tsx` |
| 함수/변수 | camelCase | `sendEmail`, `isVisible`, `handleClick` |
| 상수 | UPPER_SNAKE_CASE | `API_ENDPOINT`, `MAX_WIDTH` |
| 폴더 | lowercase | `components/`, `public/` |

## 주석 작성

```typescript
// ❌ 불필요한 주석 (코드 자체가 명확함)
const name = "John"; // name 변수 선언

// ✅ 필요한 주석 (왜 하는지 설명)
// Gmail 앱 비밀번호 사용 (일반 비밀번호 불가)
const SMTP_PASS = process.env.SMTP_PASS;

// ✅ 복잡한 로직 설명
// 이메일 발송 전 3초 대기 (SMTP 안정성)
await new Promise(resolve => setTimeout(resolve, 3000));
```

## Git 커밋 메시지

```
[타입] 제목 (최대 50자)

- feat: 새로운 기능
- fix: 버그 수정
- docs: 문서 수정
- style: 코드 포맷팅 (기능 변화 없음)
- refactor: 코드 리팩토링
- perf: 성능 개선

예시:
feat: 문의 폼에 전화번호 필드 추가
fix: Hero 섹션 모바일 반응형 버그 수정
docs: 개발 가이드 업데이트
```

## 팁

- 불필요한 라이브러리 추가 금지
