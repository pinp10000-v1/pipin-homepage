# 배포 가이드

## 배포 구조

```
로컬 PC (VS Code)
    ↓ git push
GitHub 저장소
    ↓ 자동 감지
Vercel (서버 호스팅)
    ↓ 도메인 연결
pip-biz.com (실제 홈페이지)
```

## 배포 3단계

### 1단계: 로컬 수정 및 테스트
```bash
npm run dev  # localhost:3000에서 확인
```

### 2단계: GitHub에 올리기
```bash
git add .
git commit -m "feat: 메뉴 추가" # 또는 fix/docs/style 등
git push origin master
```

### 3단계: 자동 배포 (Vercel)
- `git push` 후 Vercel이 자동 감지
- 약 1~2분 후 https://www.pip-biz.com 에 반영
- 배포 상태는 Vercel 대시보드에서 확인 가능

## 배포 실패 시

1. **Vercel 대시보드 확인**
   - https://vercel.com/dashboard
   - 프로젝트 선택 → 최근 배포 클릭

2. **빨간색 로그 확인**
   - 에러 메시지 전체 복사

3. **로그를 Claude Code에 공유**
   - 해결 방법 제시

## 배포 제한사항

- `.gitignore` 파일에 등록된 것들은 배포 안 됨
- `.env.local`은 Vercel 환경변수로 별도 설정 필요
- `node_modules/`는 Vercel에서 자동 생성
