# 배포 가이드

> 도메인: https://www.pip-biz.com  
> GitHub: https://github.com/pinp10000-v1/pipin-homepage

## 배포 구조

```
로컬 PC (VS Code + Claude Code)
    ↓ git push
GitHub (master 브랜치)
    ↓ 자동 감지 (약 1~2분)
Vercel (서버 호스팅)
    ↓ 도메인 연결
pip-biz.com (실제 홈페이지)
```

## 배포 3단계

### 1단계: 로컬 빌드 검증

```bash
npm run build
# ✓ Compiled successfully 확인
# TypeScript 에러 0개 확인
```

### 2단계: GitHub에 올리기

```bash
git add .
git commit -m "feat: 기능 설명"
git push origin master
```

### 3단계: Vercel 자동 배포

- `git push` 후 Vercel이 자동 감지
- 약 1~2분 후 https://www.pip-biz.com 반영
- 배포 상태: https://vercel.com/dashboard

## 커밋 메시지 규칙

| 타입 | 용도 | 예시 |
|------|------|------|
| `feat` | 새 기능 | `feat: 카운터 애니메이션 추가` |
| `fix` | 버그 수정 | `fix: 이미지 로딩 오류 수정` |
| `refactor` | 코드 정리 | `refactor: useReveal 공통 hook 추출` |
| `docs` | 문서 수정 | `docs: 배포 가이드 업데이트` |
| `style` | 스타일만 수정 | `style: Hero 여백 조정` |

## 배포 실패 시

1. **Vercel 대시보드 확인**
   - https://vercel.com/dashboard → 프로젝트 → 최근 배포
   - 빨간색 로그 확인

2. **로컬에서 먼저 확인**
   ```bash
   npm run build
   # 에러 메시지 확인 후 수정
   ```

3. **재배포**
   ```bash
   git add .
   git commit -m "fix: 빌드 오류 수정"
   git push origin master
   ```

## 환경변수 관리

- `.env.local` → 로컬 전용 (git 제외됨)
- Vercel 배포 환경변수 → Vercel 대시보드 → Settings → Environment Variables
- `.env.local`과 동일한 값을 Vercel에 별도 설정 필요

## 주의사항

- `pipin.dedyn.io` 주소 사용 금지 (미사용 도메인)
- 운영 도메인은 **pip-biz.com** 만 사용
- `.gitignore` 등록 파일은 배포 안 됨 (`.env.local`, `node_modules/` 등)
