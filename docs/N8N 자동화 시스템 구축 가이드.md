# N8N 자동화 시스템 구축 가이드
## "부동산/경제/금융 인사이트 뉴스" 자동 생성 및 배포

---

## 📋 시스템 개요

**목표**: 매일 부동산·경제·금융 관련 뉴스를 수집 → 요약 → AI 이미지 생성 → 블로그 게시 → 홈페이지 노출

**주요 구성요소**:
1. 뉴스 수집 (RSS/API)
2. Claude로 요약 및 인사이트 생성
3. Claude로 대표이미지 AI 생성
4. 블로그 시스템에 자동 게시
5. 홈페이지 "인사이트 뉴스" 섹션 노출

---

## 🔄 워크플로우 구조

### 1단계: 뉴스 수집 (매일 00:00 실행)

**트리거**: Cron - 매일 오전 12시 (또는 원하는 시간)

**수집 소스**:
- **부동산**: 
  - 부동산114 뉴스 RSS
  - 네이버 부동산 뉴스 API
  - 조선일보 부동산 섹션
  
- **경제**:
  - 이데일리 경제 뉴스 RSS
  - 파이낸셜뉴스 API
  - 블룸버그 경제지표
  
- **금융**:
  - 머니투데이 금융 뉴스 RSS
  - 펀드온라인 시장정보
  - 한국은행 금리 정보

**N8N 노드**:
```
[Cron Trigger]
    ↓
[HTTP Request - RSS 피드 1]
[HTTP Request - RSS 피드 2]
[HTTP Request - RSS 피드 3]
    ↓
[데이터 병합 및 중복 제거]
    ↓
[상위 3-5개 뉴스 필터링]
```

---

### 2단계: 뉴스 요약 및 인사이트 생성 (Claude API)

**처리**: 수집된 각 뉴스별로 실행

**Claude 프롬프트**:
```
당신은 부동산/경제/금융 전문가입니다.
다음 뉴스를 100자 내로 요약하고, 부동산 업계에 미치는 영향을 한 문장(50자)으로 설명하세요.

뉴스: [뉴스 제목 및 본문]

요약:
영향:
```

**출력 형식**:
- `title`: 뉴스 제목
- `summary`: 100자 요약
- `impact`: 영향 분석
- `category`: 부동산/경제/금융

**N8N 노드**:
```
[각 뉴스 항목 루프]
    ↓
[Claude API 호출 - 요약]
    ↓
[응답 데이터 정리]
    ↓
[데이터베이스 저장]
```

---

### 3단계: AI 대표이미지 생성

**처리**: 각 뉴스별 이미지 생성

**방법 A: Claude Vision + 외부 이미지 API**
- 요약 텍스트 기반 이미지 프롬프트 생성
- 외부 API (Unsplash, Pexels)에서 관련 이미지 검색
- 또는 DALL-E, Midjourney API 활용

**방법 B: 템플릿 기반 이미지 생성**
- 카테고리별 배경 템플릿 준비
- Claude로 텍스트 오버레이 생성
- 이미지 생성 라이브러리(PIL/Pillow)로 합성

**추천 설정**:
```
카테고리별 색상:
- 부동산: 파란색 계열
- 경제: 초록색 계열
- 금융: 주황색 계열

이미지 크기: 1200x630px (SNS 최적화)
포맷: JPG (빠른 로딩)
```

**N8N 노드**:
```
[요약 텍스트 입력]
    ↓
[Claude - 이미지 프롬프트 생성]
    ↓
[이미지 생성 API 호출]
    ↓
[생성된 이미지 저장 (AWS S3/로컬)]
    ↓
[이미지 URL 데이터베이스 저장]
```

---

### 4단계: 블로그 자동 게시

**플랫폼**: 현재 블로그 시스템 (Markdown/CMS)

**게시 프로세스**:

1. **마크다운 파일 생성**:
```markdown
---
title: [뉴스 제목]
date: [현재 날짜]
category: [부동산/경제/금융]
image: [생성된 이미지 URL]
excerpt: [요약]
---

## 뉴스 요약
[요약 내용]

## 부동산 업계 영향
[영향 분석]

## 원본 뉴스
[원본 링크]
```

2. **자동 게시**:
   - 로컬 블로그: 파일 시스템에 MD 파일 저장
   - 외부 CMS (Medium, Notion 등): API를 통해 게시
   - WordPress: REST API 활용

**N8N 노드**:
```
[마크다운 파일 생성]
    ↓
[파일 저장 (로컬 또는 클라우드)]
    ↓
[CMS API - 포스트 생성]
    ↓
[게시 완료 알림]
```

### 5단계: 홈페이지 노출

**HomPage NewsInsight 섹션**:

1. **데이터 소스**: 
   - 생성된 뉴스 데이터 (DB 또는 JSON 파일)
   - 최신 5개 뉴스만 표시

2. **컴포넌트**: `components/NewsInsight.tsx` 수정

```tsx
// 예시
const news = [
  {
    id: 1,
    title: "서울 아파트값 0.5% 상승",
    summary: "최근 강남구 신규 분양 물량 부족으로...",
    impact: "향후 3개월 강남권 분양 시장 강세 예상",
    category: "부동산",
    image: "/news/2026-04-07-01.jpg",
    link: "/blog/article-slug",
  },
  // ... 더 많은 항목
]
```

3. **UI**:
   - 카테고리별 색상 구분
   - 대표이미지 표시
   - 클릭 시 블로그 본문 이동
   - "더보기" 링크로 블로그 이동

## 🔧 N8N 워크플로우 설정

### 필요한 API 키
```
1. Claude API 키 (Anthropic)
2. 뉴스 API 키 (Optional - 전문 서비스)
3. 이미지 생성 API 키 (Unsplash/Pexels 등)
4. 블로그 CMS API 키
5. 클라우드 스토리지 키 (이미지 저장용)
```

### 스케줄 설정
```
- 뉴스 수집: 매일 06:00 (또는 원하는 시간)
- 이미지 생성: 수집 후 자동 실행
- 블로그 게시: 이미지 생성 완료 후
- 홈페이지 업데이트: 게시 완료 후
```

### 에러 처리
```
- API 실패 시 재시도 (최대 3회)
- 실패 알림 메일 발송
- 수동 검수 대기 상태 설정 (Optional)
```

## 💾 데이터베이스 스키마

```sql
CREATE TABLE news_articles (
  id INT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(255),
  summary TEXT,
  impact TEXT,
  category ENUM('부동산', '경제', '금융'),
  original_url VARCHAR(500),
  image_url VARCHAR(500),
  blog_post_slug VARCHAR(255),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  published_at TIMESTAMP,
  status ENUM('draft', 'published', 'archived')
);
```

## 🚀 실행 방법

### 수동 실행
1. N8N 콘솔 접속
2. 해당 워크플로우 선택
3. "Execute Workflow" 클릭
4. 결과 확인

### 자동 실행
1. Cron 트리거 설정
2. 일정 시간에 자동 실행
3. Webhook으로 외부 트리거 가능

### 모니터링
1. N8N 대시보드에서 실행 기록 확인
2. 로그로 에러 추적
3. Slack/이메일 알림 설정

## 📌 주의사항

1. **API 비용**: Claude API 사용에 따른 비용 발생 (월 $5-50)
2. **이미지 라이선스**: 외부 이미지 사용 시 라이선스 확인
3. **뉴스 저작권**: 원본 뉴스 링크 반드시 표기
4. **일일 실행 한계**: 너무 많은 뉴스 처리 시 API 한계 주의

## 📞 문제 해결

| 문제 | 해결책 |
|------|--------|
| 뉴스가 수집되지 않음 | RSS 피드 URL 확인, 네트워크 확인 |
| Claude 응답 오류 | API 키 확인, 요청 토큰 한계 확인 |
| 이미지 생성 실패 | API 한계, 프롬프트 최적화 |
| 블로그 게시 실패 | CMS API 키, 경로 설정 확인 |
